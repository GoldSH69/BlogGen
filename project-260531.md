# 🚀 AffiliWrite AI 프로젝트 개발 가이드라인 & 연동 지침 (260531)

이 파일은 **AffiliWrite AI** 프로젝트의 핵심 수집 엔진 및 UI 고도화 이력과 Antigravity 코딩 어시스턴트가 작동할 때 절대 엄수해야 하는 **대원칙 지침**을 보존하여 토큰 소모를 예방하고 일관된 협업 무결성을 유지하기 위한 공식 영구 문서입니다.

## 🚨 [최우선 의무] 소스 코드 수정 및 편집 대원칙

후속 작업에 참여하는 모든 Antigravity 에이전트는 아래의 협업 개발 원칙을 **200% 철저히 준수**하십시오. 이 원칙은 사용자의 명시적 제어 권한을 존중하고 토큰을 아끼기 위한 준엄한 대원칙입니다.

1. **승인 전 소스코드 수정 절대 금지**:
   - 어떠한 버그 핫픽스나 기능 고도화 요청이 들어오더라도, **사전에 코드를 단 1줄이라도 임의로 수정/편집/추가/삭제하지 마십시오.** (로컬 수정 후 '승인 대기' 상태로 대기하는 행위 조차도 절대 불가합니다.)
2. **텍스트 기반의 사전 협의 및 확정**:
   - 이상 현상의 근본 원인과 해결 방안을 **오직 텍스트 응답으로만** 정밀하게 제안하십시오.
   - 제안된 기획 사양에 대해 사용자와 의견을 나누어 내용을 완벽하게 **"확정"**받아야 합니다.
3. **명시적 승인 후 구현 개시**:
   - 사용자의 명시적인 **"승인"**, **"수정해라"**, **"반영해라"**라는 텍스트 사인이 떨어진 후에만 비로소 소스 코드 파일 수정 툴을 최초로 기동하여 작업을 집도하십시오.
4. **배포/푸시 컨펌 절차**:
   - 수정을 마친 후 컴파일 빌드가 통과하면 즉시 푸시하지 말고, 구현된 변경 사항을 사용자에게 팩트 위주로 1차 보고하십시오.
   - 사용자가 검토를 마친 후 **"푸시해라"** 또는 **"배포해라"**라고 명시적 최종 사인을 주었을 때만 비로소 `git push origin main` 명령을 기동하여 실서버 릴리즈를 배포하십시오.

## 📂 3대 그룹 트렌드 수집 설정 완전 격리 아키텍처

트렌드 피드 수집 및 가공 엔진은 아래와 같이 **3개 카테고리별로 옵션이 상호 간섭 없이 철저하게 개별 기동**하도록 분할 설계 및 적용되었습니다.

### 1. 설정 구조 계층화 (`trend-rules.json`)
```json
{
  "myInterest": {
    "keywords": ["건강 정보", "경제 재테크", "IT 트렌드"],
    "sources": { "naverBlog": true, "naverNews": true, "naverShopping": false },
    "filtering": { "minCleanScore": 80, "customBlacklist": ["공구", "마켓"], "checkAdRegex": true }
  },
  "naverHotTopic": {
    "keywords": ["다이소 꿀템", "코스트코 추천템", "가전 전자제품"],
    "sources": { "naverBlog": true, "naverNews": false, "naverShopping": false },
    "filtering": { "minCleanScore": 90, "customBlacklist": ["홍보", "체험단"], "checkAdRegex": true }
  },
  "realtimeHotIssue": {
    "keywords": ["AI 인공지능", "신제품 출시"], 
    "sources": { "naverBlog": true, "naverNews": true, "naverShopping": true },
    "filtering": { "minCleanScore": 75, "customBlacklist": ["광고", "협찬"], "checkAdRegex": true }
  },
  "scheduler": {
    "intervalHours": 6
  }
}
```

### 2. 컴포넌트별 믹싱 사양
- **`TrendSettingsPanel.jsx` (트렌드 제어 모달)**:
  - 설정 모달 내부에 `📌 내 관심사`, `🔥 네이버 핫토픽`, `⚡ 실시간 핫이슈` 탭 바를 렌더링합니다.
  - 각 탭 선택 시 해당하는 격리 영역의 키워드 및 채널 범위, 클린 필터 임계치를 상호 간섭 없이 개별 관리 및 병합 저장합니다.
- **`trend-crawler.cjs` (백엔드 수집 엔진)**:
  - 1그룹 '내 관심사' 수집기는 `myInterest` 옵션 기반 기동.
  - 2그룹 '네이버 핫토픽' 수집기는 `naverHotTopic` 옵션 기반 기동 (RSS 공지 덤프를 전면 제거하고, 네이버 검색 API의 `sort=sim` 유사도 노출 정렬을 적용해 조회수 최상단 글들을 정교하게 믹싱 수집).
  - 3그룹 '실시간 핫이슈' 수집기는 `realtimeHotIssue` 옵션 기반 기동.

## 🛠️ 260531 추가 고도화 및 보완 수립 명세

파트너님의 피드백에 기반하여 아래의 고고도 사용성 개선 및 무결성 제어 장치를 추가 이식 완료하였습니다.

1. **테마 호환성 완전 극대화 패치 (`ThumbnailKit.jsx`)**:
   - 기존에 어두운 배경색으로 하드코딩되어 라이트 모드 시 겉돌던 썸네일 킷 카드를 전면 CSS 테마 변수(`var(--bg-surface-solid)`, `var(--border-color)`, `var(--text-primary)`) 사양으로 드레스업하여 듀얼 모드에 100% 동화시켰습니다.
2. **AI 날짜 왜곡(2024년 Hallucination) 차단 (`gemini.js`)**:
   - AI 프롬프트 초입에 작성 기준일 `2026-05-31` 오늘 날짜를 강제 변수 주입하고, Frontmatter date 및 mdx 파일명 생성 시 절대 과거 년도를 지어내지 말라는 "날짜 강제 규정"을 철저히 심어 과거 날짜 오류를 예방하였습니다.
3. **WebP 다운로드 파일명 기존 이름 보존 (`ThumbnailKit.jsx`)**:
   - 1200x514 스마트 크롭 및 WebP 다운로드 시 원래 업로드한 이미지 파일명(예: `sunset.png`)을 소중히 추적하여, 파일명 손실 없이 `sunset.webp` 확장자 전환 스와프 형태로 저장되도록 다운로드 명세을 고도화하였습니다.
4. **네이버 블로그 본문 내 이모티콘 금지령 (`gemini.js`)**:
   - 구글 애드센스 승인 품질 심사에 나쁜 영향(스팸성 판정 등)을 주는 현상을 원천 방어하고자, 소제목(📌 등) 이외의 나머지 일반 문장 텍스트에는 어떠한 형태의 이모지 및 이모티콘 사용도 100% 전면 금지하는 강력 규칙을 헌법화했습니다.
5. **스마트 복사 코드박스 상시 반투명 연보라색 우측 스크롤 노출 (`OutputTabs.jsx`)**:
   - 코드박스 body에 `overflow-y: scroll`을 상시 기동하고, 바이올렛 테마에 녹아나는 미려한 반투명 커스텀 스크롤바 스킨을 주입하여 아래에 원고 내용이 더 존재함을 마우스 조작 없이도 단박에 알 수 있도록 직관화했습니다.
6. **[신규 기능] 깃허브 피드 이슈 즉시 제외(닫기) 기능 (`TrendDiscoveryFeed.jsx` & `github.js`)**:
   - 피드 내 카드를 보다가 원치 않는 스팸 기사가 있을 시, 카드 우측 상단 뱃지 옆의 빨간색 **휴지통(Trash2)** 단추를 클릭해 실시간으로 깃허브 이슈를 `closed` 처리하고 피드 목록에서 즉각 배제할 수 있는 권한을 탑재했습니다.

## 🧪 테스트 방법 및 예상 결과

### 1. UI 및 연동 테스트 방법 (프론트엔드)
1. **모달 진입 및 초기 동기화 검증**:
   - BlogGen 대시보드 우측 상단의 `트렌드 수집 설정` (⚙️) 버튼을 클릭합니다.
   - 모달 내부 상단에 `📌 내 관심사`, `🔥 네이버 핫토픽`, `⚡ 실시간 핫이슈` 탭이 premium 듀얼 테마 스타일에 부합하게 노출되는지 확인합니다.
   - 최초 진입 시 GitHub의 `trend-rules.json` 설정에 따라, 각 탭별로 상이한 키워드와 클린도 점수가 표시되는지 점검합니다.
2. **개별 옵션 격리 및 병합 저장 테스트**:
   - `📌 내 관심사` 탭에서 키워드 하나를 삭제 또는 추가하고, 최소 클린 스코어를 85점으로 변경합니다.
   - `🔥 네이버 핫토픽` 탭으로 전환하여 방금 변경한 내 관심사의 상태가 이 탭에 침범하지 않았는지(원래 키워드와 클린도 90점 유지) 확인합니다.
   - `⚡ 실시간 핫이슈` 탭도 개별 동작함을 검증한 뒤, 우측 하단의 `[트렌드 제어 설정 커밋]` 버튼을 누릅니다.
3. **결과 확인**:
   - 설정 저장이 완료되었다는 확인 메시지가 뜬 뒤, GitHub 저장소의 `trend-rules.json` 파일이 3분할 3-Tier 아키텍처에 부합하는 병합 데이터 구조로 우아하게 커밋 업데이트 되었는지 확인합니다.

### 2. 백엔드 수집 엔진 구동 테스트 방법 (GitHub Actions / Local)
로컬 터미널 혹은 GitHub Action runner 환경에서 아래 명령어를 기동하여 수집 무결성을 정밀 측정합니다.

```powershell
# 네이버 OpenAPI Key 환경변수 주입 후 크롤러 수동 실행
$env:NAVER_CLIENT_ID="[CLIENT_ID]"
$env:NAVER_CLIENT_SECRET="[CLIENT_SECRET]"
$env:GITHUB_TOKEN="[GITHUB_TOKEN]"
$env:GITHUB_REPOSITORY="[USER]/[REPO]"
node scripts/trend-crawler.cjs
```

### 3. 예상 결과 및 통과 기준 (Expected Results)
* **1그룹 (📌 내 관심사)**:
  * 로그 창에 `[1그룹] 내 관심사 트렌드 수집 시작...` 콘솔이 출력됩니다.
  * `myInterest`에 등록된 키워드(예: 건강 정보 등)로만 네이버 API 검색을 수행하고, 80점 이상의 클린 피드들만 고밀도로 선별 수집됩니다.
* **2그룹 (🔥 네이버 핫토픽)**:
  * 로그 창에 `[2그룹] 네이버 블로그 실시간 핫토픽 인기글 수집 시작...` 콘솔이 출력됩니다.
  * 네이버 RSS 공지글 도배 현상이 완전히 영구 차단됩니다.
  * 대신 `naverHotTopic.keywords`에 등록된 키워드(예: 다이소 꿀템 등)로 네이버 블로그를 유사도(`sort=sim`) 검색하여, 90점 이상의 압도적인 파워 포스팅 상위 노출글만 선발 수집됩니다.
* **3그룹 (⚡ 실시간 핫이슈)**:
  * 로그 창에 `[3그룹] 구글 RSS & 쇼핑베스트 실시간 핫이슈 수집 시작...` 콘솔이 출력됩니다.
  * 구글 실시간 인기 급상승 검색어와 쇼핑 베스트 100 상품명이 자동 파싱됩니다.
  * 수집된 실시간 키워드를 활용해 구글 뉴스 실시간 기사 및 네이버 API 검색 결과가 기동되며, 75점 이상의 유효 피드들만 적재됩니다.
* **통합 결과 및 중복 방지**:
  * 각 그룹에서 선발된 피드 카드가 제목/링크 기준으로 2중 중복 제거 처리를 완벽하게 통과합니다.
  * 중복이 전혀 없이 깔끔하게 걸러진 최종 트렌드 카드 후보들에 한해서만 상세 본문 크롤링이 안전하게 기동됩니다.
  * 등록되지 않은 새로운 글들만 GitHub Repository Issue 탭에 `[트렌드] ...` 타이틀로 정확하게 단 한 번씩만 발행됩니다.
