## 📂 카테고리별 인기글 실시간 수집 아키텍처 및 상세 사양

사용자님의 명확한 지침에 따라 기존의 키워드 매핑식 레이더를 폐지하고, **네이버 공식 4대 대분류 및 31개 중분류 카테고리별 실시간 최상위 인기 트래픽 글들을 사냥하는 엔진**을 구현합니다.

### 1. 설정 구조 확장 사양 (`trend-rules.json` 동기화)
- 기존 `naverBlogRadar` 항목을 **`naverCategoryPopular`** 객체 구조로 갱신하여 클라우드에 영구 백업 및 동기화합니다:
```json
{
  "naverCategoryPopular": {
    "categories": [30, 33, 29, 14], 
    "sources": { "naverBlog": true, "naverNews": false, "naverShopping": false },
    "filtering": { "minCleanScore": 80, "customBlacklist": ["광고", "체험단"], "checkAdRegex": true, "maxAgeDays": 30 }
  }
}
```

### 2. 컴포넌트별 구현 완료 사항
- **`TrendSettingsPanel.jsx` (트렌드 설정 모달) [수정 완료]**:
  - 4번째 탭의 이름을 `📈 카테고리 인기글`로 변경하였습니다.
  - 키워드 입력 칸 대신, **네이버 공식 4대 분류 체계**에 기반한 그리드형 카테고리 체크박스 선택 패널을 구현하였습니다.
  - 사용자가 체크박스를 활성화/비활성화할 때마다 해당 카테고리의 고유 `seq` 번호가 `categories` 배열에 자동으로 추가 및 제거되도록 동적 핸들러를 완벽하게 연동하였습니다.
- **`TrendDiscoveryFeed.jsx` (트렌드 피드) [수정 완료]**:
  - 4번째 피드 세그먼트 버튼을 `📈 카테고리 인기글`로 변경하였습니다.
  - 수집 데이터 파싱 엔진에 **공감수(`sympathyCnt`)** 및 **댓글수(`commentCnt`)** 추출 정규식을 완비하였습니다.
  - 카드 UI의 중간 타이틀 아래 영역에 작성자(블로거) 정보와 함께 **`❤️ 125`**, **`💬 14`** 형태로 실시간 트래픽 지표 배지를 추가하여 정보의 프리미엄 디자인 감각을 극대화하였습니다.
- **`scripts/trend-crawler.cjs` (백엔드 크롤러) [수정 완료]**:
  - `trend-rules.json`의 `naverCategoryPopular` 설정을 유연하게 읽어옵니다.
  - 활성화된 각 카테고리 `seq`에 대해 비공개 AJAX API인 `https://section.blog.naver.com/ajax/DirectoryPostList.naver`를 안전하게 호출합니다.
  - 네이버의 JSON 변조 방어자(`)]}',`)를 정규식으로 안전하게 제거한 후 JSON 데이터를 파싱하여 최상위 트래픽 포스트를 추출합니다.
  - 대가성 광고글(원고료 지원 등) 스팸 클린 필터링을 실행하고, 통과된 A급 글들의 모바일 본문 주소로 접근하여 **본문 알맹이 스냅샷(`scrapeFullText`)**을 통째로 긁어옵니다.
  - 깃허브 이슈 발행 시 `- **수집 그룹**: 📈 네이버 카테고리 인기글`, 작성자 항목에 `닉네임 (공감 X / 댓글 Y)` 포맷을 탑재하여 완벽하게 저장합니다.

## 🚨 [대원칙] 100% 무료 플랜 전용 다단계 폴백(Fallback) 체인 운용 규칙 (260602 추가)

API 서버 부하나 할당량 제한(Rate Limit)으로 인한 작성 오류("This model is currently experiencing high demand...")를 실시간으로 우회하기 위해, 100% 무료 플랜(Free Tier)으로 보장되는 안전 모델군으로 구성된 **다단계 폴백 체인**을 상시 가동하고 엄격히 관리해야 합니다.

1. **사용 가능한 100% 무료 플랜 모델군 명세 (Free-Tier Certified Models)**:
   - **`gemini-3.5-flash`** (1순위): 최신 3.5세대 Flash 모델로 뛰어난 한국어 가독성과 고속 처리를 무상으로 제공합니다.
   - **`gemini-2.5-flash`** (2순위): 검증된 2.5세대 Flash 모델로 오랜 시간 축적된 블로그 글 작성 무결성을 지닌 기본 모델입니다.
   - **`gemini-2.0-flash`** (3순위): 안정적인 2.0세대 Flash 백업 모델입니다.
   - **`gemini-3.1-flash-lite`** (4순위): 3.1세대 최신 Lite 모델로, 대형 서버 혼잡 시간대에도 부하가 거의 없으며 가장 높은 트래픽 우회율을 가집니다.
   - **`gemini-2.5-flash-lite`** (5순위): 2.5세대 Lite 백업 모델입니다.

2. **유료(Pro 계열) 모델의 강력 배제 규칙**:
   - Google AI Studio 정책에 따라 **Pro 계열 모델(`gemini-2.5-pro`, `gemini-3.1-pro-preview` 등)은 무료 플랜에서 제외되어 유료 결제(Pay-as-you-go) 계정에서만 호출 가능**합니다.
   - 무료 API 키 환경에서 Pro 계열 모델을 호출할 시 즉각 호출 실패 에러('Billing not enabled')를 유발하여 폴백 체인 전체가 파괴되므로, **폴백 리스트에 Pro 계열 모델명을 기재하는 행위를 절대 엄금**합니다.

3. **폴백 구현 가이드라인**:
   - `executeWithFallback` 메서드를 통해 지정된 무료 모델 체인을 순차적으로 루프 순회하며 호출해야 합니다.
   - 특정 단계에서 오류 발생 시 해당 모델명과 에러 원인을 `console.warn`에 남겨 투명하게 로그 추적이 가능하게 조치하십시오.
   - 모든 대체 모델 체인이 실패했을 때만 비로소 예외 처리를 화면으로 전달하되, 이때 가장 마지막으로 실패한 모델명 접두사(예: `[gemini-2.5-flash]`)를 에러 메시지에 함께 명기하여 사용자가 문제를 즉시 진단할 수 있도록 구조화하십시오.
